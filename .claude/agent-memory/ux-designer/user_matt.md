Matt (schwamat@gmail.com) is the builder and primary user of Script Kitties Editor.

- **Audience is one — for now.** The tool is built for Matt, who knows the `worlds-cpp` game's data model cold. **Modders are a later audience.** So: design for fluency now (don't over-explain the data model), but don't bake in assumptions that would block a less-expert modder later — especially in the API reference / discoverability surfaces.
- Works in flows: asks for the plan before any building ("before doing anything, tell me your plan"). Lead with the plan, surface decisions and open questions, get a green light before producing artifacts.

**How to apply:** Tune teaching level to an expert for v1, but treat zero-tutorial discoverability as a forward investment for the modder audience. Always present a plan before creating files.
